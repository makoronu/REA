# HTML構造調査

## やること
対象サイトの検索結果ページと物件詳細ページのHTML構造を調査する。

## 調査対象ページ

### 1. 検索結果ページ（一覧）
- 物件種別ごとに1つずつサンプルURL取得（土地・戸建・マンション）
- 以下を調査：
  - 物件一覧のコンテナ要素（class/id）
  - 各物件カードの要素（class/id）
  - 物件詳細へのリンク要素（href パターン）
  - ページネーション要素（「次へ」ボタンの構造）
  - 1ページあたりの物件数

### 2. 物件詳細ページ
- 物件種別ごとに1つずつサンプルURL取得
- 以下を調査：
  - 物件名/タイトルの要素
  - 価格の要素
  - 物件情報テーブルの構造（dl/dt/dd or table/tr/th/td）
  - 各フィールドのラベル文言（「価格」「土地面積」「所在地」等）
  - 会社情報の要素
  - 画像の要素（将来用、今は取得しない）

## 調査方法
```bash
# Seleniumでページ取得してHTML保存
# または curl で静的ページの場合
curl -s -o /tmp/{site}_{page_type}.html "https://..."
```

## 記録フォーマット
```
【サイト】: {サイト名}
【一覧ページ構造】:
  - コンテナ: div.property-list
  - 物件カード: div.property-card
  - リンク: a.property-link[href]
  - ページネーション: div.pagination > a.next
  - 物件数/ページ: 20件

【詳細ページ構造】:
  - タイトル: h1.property-title
  - 価格: span.price
  - 情報テーブル: dl形式 / table形式
  - テーブルラベル一覧:
    - "価格" → sale_price
    - "所在地" → address
    - "土地面積" → land_area
    - ...
  - 会社情報: div.company-info
  - Selenium必要: はい / いいえ（JavaScript動的レンダリングの有無）
```
