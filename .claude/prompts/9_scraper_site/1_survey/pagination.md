# ページネーション方式特定

## やること
一覧ページのページ遷移方式を特定し、全ページ巡回のロジックを決定する。

## 確認項目

### ページネーション方式
- [ ] URLパラメータ型（`?page=2`, `?p=2`, `?offset=20`）
- [ ] パスベース型（`/page/2/`）
- [ ] JavaScript動的読み込み（「もっと見る」ボタン、無限スクロール）
- [ ] その他

### URL パターン
```
1ページ目: {base_url}
2ページ目: {base_url}?page=2
3ページ目: {base_url}?page=3
```

### 終了判定
- [ ] 「次へ」リンクが存在しない
- [ ] 物件数が0件
- [ ] max_pages に到達
- [ ] 取得URLが前回と重複

### 地域フィルタ
- 検索URL内でどのパラメータ/パスで地域を指定するか
- 北海道全域、札幌市、特定区の URL パターン

## 記録フォーマット
```
【サイト】: {サイト名}
【ページネーション方式】: URLパラメータ型 / パスベース型 / JS動的型
【URLパターン】:
  土地: https://.../{area}/land/?page={n}
  戸建: https://.../{area}/kodate/?page={n}
  マンション: https://.../{area}/mansion/?page={n}

【1ページあたり件数】: 20件
【終了判定】: 次へリンク非存在
【最大ページ数上限】: 50ページ（安全策）
```
