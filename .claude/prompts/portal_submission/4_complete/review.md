# レビュー・承認

## やること

1. 成果物一覧をユーザーに提示
2. 設定ファイルのレビュー依頼
3. 運用手順の確認

## 成果物一覧

```
portal_configs/{portal_name}/
  ├── portal.json      # ポータル基本情報
  ├── auth.json        # 認証設定
  ├── form.json        # フォーム定義（セレクタ、バリデーション）
  ├── mapping.json     # マッピング定義
  └── transform.json   # 変換ルール

docs/
  └── portal_{portal_name}_diff.md  # 仕様書との差異（あれば）
```

## 運用手順

```
1. 入稿スクリプト起動
2. ログイン（手動介入必要な場合は待機）
3. 物件選択
4. 自動入力実行
5. 【停止】登録ボタン直前で待機
6. 【ユーザー操作】内容確認 → 登録ボタン押下
7. 完了確認
```

## チェックリスト

- [ ] 全設定ファイル確認済み
- [ ] マッピング漏れなし
- [ ] テスト結果PASS
- [ ] 運用手順理解済み
- [ ] **登録ボタンはユーザーが押すことを確認**
