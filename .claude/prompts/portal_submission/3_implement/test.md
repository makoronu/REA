# テスト実行

## やること

1. テスト物件データで入稿テスト
2. 各フェーズの動作確認
3. エラーケースの確認

## テスト手順

### 1. ログインテスト
```
- 認証成功するか
- 手動介入が必要な場合、適切に待機するか
- セッション維持できるか
```

### 2. フォーム入力テスト
```
- 全項目に値が入るか
- セレクタが正しいか
- 動的要素が処理できるか
```

### 3. バリデーションテスト
```
- 必須チェックが機能するか
- 形式チェックが機能するか
- エラー表示を検出できるか
```

### 4. 画像アップロードテスト
```
- 画像選択できるか
- サイズ・形式変換が機能するか
- 複数画像の順序が正しいか
```

### 5. 登録待機テスト
```
- 登録ボタン直前で停止するか
- ユーザーへの通知が出るか
- ユーザー操作後に完了検知できるか
```

## エラーケース

| ケース | 期待動作 |
|--------|----------|
| ログイン失敗 | リトライ後、手動介入要求 |
| セレクタ見つからない | 停止、エラー報告 |
| バリデーションエラー | 停止、エラー内容表示 |
| 画像アップロード失敗 | リトライ後、スキップ可否確認 |

- [ ] エラー系テストPASS
- [ ] 登録待機動作確認済み
