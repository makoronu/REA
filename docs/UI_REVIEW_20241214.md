# UI精査レポート（2024-12-14）

全10物件種別のスクリーンショットを精査した結果 + ユーザー判断。

---

## システムの位置づけ（重要）

**このシステムは「査定から決済まで一貫して使う」ことを前提とする。**
ホームページ掲載だけでなく、不動産業務全体をカバーする。

## 2. 設計変更（要実装）

### 2.1 取引態様と元請会社の連動

**現状**: 元請会社セクションが常に表示

**変更後**:
1. 「取引態様」を独立させてラジオボタンに
2. 選択肢に「仲介」を追加
3. 「仲介」選択時のみ元請会社入力欄を表示
4. 空欄 = 自社物件と判定

### 2.2 ステータス名の変更

**現状**: 「販売」という名前

**変更後**: より業務フローに沿った名称に

| 新ステータス | 説明 |
|-------------|------|
| 査定中 | 査定依頼を受けて評価中 |
| 保留 | 一時的に活動停止 |
| 販売準備 | 写真撮影、資料作成中 |
| 販売中 | アクティブに販売活動中 |
| 商談中 | 買主候補と交渉中 |
| 成約済み | 契約締結完了 |
| 販売終了 | 販売活動終了（理由不問） |
| 取り下げ | 売主都合で中止 |
| 他決 | 他社で成約 |

**ステータス名の提案**:
- 「物件ステータス」
- 「進捗ステータス」
- 「取引ステータス」
- 「案件ステータス」← **推奨**（査定から決済まで一貫管理なので）

### 2.3 物件種別による設備表示制御

**対応**: 前に作ったアドミン画面と同様に、物件種別ごとに表示する設備カテゴリを設定できるようにする

| 物件種別 | 非表示にすべき設備カテゴリ |
|---------|-------------------------|
| 土地 | 空調、キッチン、バス、トイレ、居住情報、省エネ |
| 駐車場 | 空調、キッチン、バス、トイレ、居住情報、省エネ、電車・鉄道 |

### 2.4 公開時バリデーション

- 入力時は自由に保存OK
- **「公開」ボタン押下時に必須項目チェック**
- 不備があれば公開不可、エラー表示

## 3. 検討事項（なんか考えろ）

| 問題 | 現状 | 検討方向 |
|------|------|---------|
| 設備タブが長すぎ | 13,000px | カテゴリ別アコーディオン折りたたみ |
| 学校候補リストが長い | 20件以上 | 件数制限（5件）or スクロール領域化 |
| モーダルが巨大 | 13,000px | 固定高さ + 内部スクロール |
| 設備チェックボックス多すぎ | 80項目以上 | アコーディオン + 物件種別フィルタ |

---

## 4. 残す（問題なし）

| 項目 | 理由 |
|------|------|
| 所属グループ | 支店管理に必要 |
| 自社管理番号 | 必要。形式: `会社コード-英数字`（しろくま=4690） |
| 元請会社セクション | 他社仲介で必要（条件表示に変更） |
| 媒介契約日 | 業務管理に必要 |
| 掲載開始日・確認日 | 2週間ごとの確認義務あり |
| 私道面積・私道負担 | マンションでも一応入れておく |
| 空白・余白 | 見やすい |
| ラベルサイズ | 問題なし |
| 学校候補（駐車場以外） | 放置でOK |
| 土地権利詳細（マンション） | 放置 |
| 用途地域自動取得精度 | 問題なし |
