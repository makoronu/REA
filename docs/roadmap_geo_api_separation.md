# ロードマップ: Geo API分離（3サービス化）

## 概要

モノリスのREA APIを3つの独立サービスに分離する。
戦艦大和型（巨大モノリス）から駆逐艦型（小型・機動的）への移行。

## アーキテクチャ

| サービス | ポート | 役割 | エンドポイント数 |
|----------|--------|------|-----------------|
| Core API | :8005 | 物件CRUD+認証+登記+メタデータ | ~77 |
| Geo API | :8007 | 地理情報（読み取り専用） | ~11 |
| Integration API | :8009 | Zoho+reinfolib+ポータル連携 | ~27（将来） |

## セグメント

| Seg | 内容 | 状態 |
|-----|------|------|
| 14a | Geo API新サービス作成（並行稼働） | 未着手 |
| 14b | 切り替え（nginx + Core API書き換え） | 未着手 |
| 15 | Integration API分離 | 将来 |
| 16 | Core APIクリーンアップ | 将来 |

## 設計原則

- 書き込みはCore APIに集約（分散トランザクション回避）
- Geo APIは読み取り専用
- サービス間通信はHTTP（同一サーバー内、localhostアクセス）
- 各サービスは独立して再起動可能
